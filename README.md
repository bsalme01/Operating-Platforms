# Operating-Platforms
# CS 230 Portfolio Reflection

### Briefly summarize The Gaming Room client and their software requirements. Who was the client? What type of software did they want you to design?

The client was The Gaming Room, a company that developed the game Draw It or Lose It. Their game was originally designed for Android devices, but they wanted to expand it into a web based application that could run on multiple operating systems and devices. The software needed to support multiple teams and players, ensure that game and team names were unique, and provide a reliable and scalable environment for users to play together online.

### What did you do particularly well in developing this documentation?

I think I did a good job analyzing different operating platforms and explaining how each one could support the client's needs. I was able to compare the strengths and weaknesses of Linux, Windows, Mac, and mobile devices while keeping the client's requirements in mind. I also provided clear recommendations and explained why Linux would be the best choice for hosting the application.

### What about the process of working through a design document did you find helpful when developing the code?

Creating the design document helped me understand the project before writing any code. It allowed me to organize requirements, identify potential challenges, and think about how different parts of the application would work together. Having a plan made it easier to understand the overall structure of the software and how design patterns such as the Singleton pattern could be used to meet the client's requirements.

### If you could choose one part of your work on these documents to revise, what would you pick? How would you improve it?

If I could revise one part of the project, I would spend more time expanding the security and scalability sections. While I covered the basic concepts, I think I could provide more detailed examples of how the system would handle large numbers of users and protect sensitive information. Adding more real world examples would make the design document even stronger.

### How did you interpret the user’s needs and implement them into your software design? Why is it so important to consider the user’s needs when designing?

I started by identifying the main goals and requirements provided by The Gaming Room. They wanted a system that could support multiple users, run across different platforms, and remain reliable as the application grew. I used those requirements to guide my recommendations for the operating platform, system architecture, storage management, and security features. Considering the user's needs is important because the software must solve the client's problems and provide a positive experience for the people using it.

### How did you approach designing software? What techniques or strategies would you use in the future to analyze and design a similar software application?

I approached the design process by first gathering and analyzing the requirements. Then I evaluated different technologies, operating systems, and architectural approaches that could meet those requirements. I compared the advantages and disadvantages of each option before making recommendations. In the future, I would continue using a structured design process that includes requirements analysis, system modeling, documentation, and evaluation of possible solutions before beginning development. This approach helps create software that is organized, scalable, and easier to maintain.
